# 精密 CNC 加工廠 — 加工異常根因分析 (RCA) 作業規範

## 1. 適用範圍

本規範適用於精密 CNC 加工廠的加工異常事件根因分析。涵蓋:
- 尺寸偏移(關鍵尺寸 ±5μm 以上偏離公差中心)
- 表面粗糙度劣化(Ra 值超出規格)
- 加工件批次性 NG(同 NC 程式同機台某批量化異常)
- 機台異常停機 / 主軸警報
- CMM 量測值與機上量測值不一致
- 跨機台、跨班別的良率落差

## 2. 角色與職責

| 角色 | 職責 |
|---|---|
| 線上技師 (Operator) | 第一線發現異常、執行初步排查 |
| RCA Agent / 系統 | 跨機台 / 工模 / 材料 / 環境自動診斷,提出調查計畫與結論 |
| 製造組長 | 審核 agent 結論、決定停機 / 重工 / 換料 / 補刀 |
| 模治具工程師 | 執行刀具更換、夾具校正、模具修整 |
| QA 工程師 | CMM 量測、SPC 規則維護、歷史案例庫管理 |

## 3. 標準調查流程

### Phase 1:擬定調查計畫 (Plan)

收到異常通報後,**第一個動作是擬定 3-5 步調查計畫**,而非直接查資料。每步必須包含:
- **假設**:用組長聽得懂的話描述要驗證什麼
- **動作**:預計要查的系統與資料

### Phase 2:多源診斷執行

執行計畫,使用以下工具:

#### 機台側工具
- `query_cnc_status(machine_id, window)` — 主軸轉速 / 進給 / 振動 / 溫度
- `query_tool_life(tool_id)` — 刀具累計加工時數、估算剩餘壽命
- `query_spindle_load(machine_id, window)` — 主軸負載趨勢(磨損的早期訊號)
- `query_nc_program(machine_id)` — 當前 NC 程式版本 + 上次變更時間

#### 量測 / QC 側工具
- `query_inspection_log(part_id_or_batch)` — CMM / 三次元量測紀錄
- `query_spc_trend(metric, window)` — SPC 管制圖,看趨勢與超出限
- `query_in_process_gauge(machine_id, window)` — 機上量測值(在線量測)

#### 上游 / 環境工具
- `query_material_batch(lot_id)` — 材料硬度、批次號、進料日期
- `query_fixture_log(fixture_id)` — 夾具校正紀錄、上次拆裝時間
- `query_environment(window)` — 廠房溫度 / 濕度(精密件對環境敏感)
- `query_recent_changes(machine_id, window)` — 程式 / 偏移量 / 刀具補正 最近異動

### Phase 3:歷史案例對照 (RAG)

調查時必須**明確引用歷史案例**,展示「用古鑑今」:

範例:「步驟 1:先看是不是刀具壽命到了(**參考 INC-3001,當年 T-08 球刀加工到 4.5 小時後尺寸開始飄移**)」

### Phase 4:強因果證據後下結論

呼叫 `submit_conclusion`,提供:
- root_cause:技術性根因
- business_impact:預估報廢數、重工工時、影響客戶交期
- recommended_actions:立即處置 + 預防措施

## 4. 5 大根因原型

下列為廠內歷史最常見的事故根因原型,Agent 應該優先檢驗:

### 4.1 刀具磨損 / 崩刃
- **症狀**:單一機台單一刀具加工件**尺寸逐漸偏移**或表面粗糙度劣化;同 lot 其他刀具加工件正常
- **典型徵兆**:刀具壽命接近上限、主軸負載 SPC 趨勢上升、Ra 值連續超出限
- **必查項**:`query_tool_life(tool_id)`、`query_spindle_load`、`query_inspection_log`
- **參考案例**:INC-3001(T-08 球刀到 4.5h 偏移)、INC-3009(粗加工刀崩刃)

### 4.2 熱變形 / 機台溫升
- **症狀**:**單一機台**連續運轉數小時後尺寸**漸進性**漂移,所有刀具加工件都受影響;停機冷卻後恢復
- **典型徵兆**:主軸溫度上升、廠房溫度異常、X/Y/Z 軸熱補償未啟動或失效
- **必查項**:`query_cnc_status` (溫度) + `query_environment` (廠房) + `query_in_process_gauge` (在線量測趨勢)
- **參考案例**:INC-3018(夏季空調故障熱漂移)、INC-3022(主軸保養後熱補償未開)

### 4.3 夾具鬆動 / 工件定位異常
- **症狀**:**某一批**加工件全部偏移(不是逐漸漂移,是突然偏移),其他批件正常;通常發生在拆裝夾具後第一批
- **典型徵兆**:夾具拆裝後沒做校正、定位銷磨損、夾持力不足
- **必查項**:`query_fixture_log(fixture_id)`、`query_recent_changes(machine_id)`
- **參考案例**:INC-3007(夾具拆洗後沒回校位)、INC-3015(定位銷磨耗)

### 4.4 NC 程式 / 偏移量錯誤(換班 / 程式更新後)
- **症狀**:**換班後或 NC 程式更新後**整批件全部偏移同樣的量;偏移量是固定值不是漂移
- **典型徵兆**:刀具補正未跟著新刀具更新、工件座標系設錯、CAM 重新後處理過
- **必查項**:`query_nc_program(machine_id)`、`query_recent_changes(machine_id, last_24h)`
- **參考案例**:INC-3025(夜班改 offset 沒登記)、INC-3031(CAM 重發後 G54 異動)

### 4.5 材料批次差異
- **症狀**:同 NC 程式同機台,**不同材料批次**加工結果不同;表面狀態 / 切削音 / 屑形變化
- **典型徵兆**:新批次材料硬度不同、表面氧化層厚度差異、含碳量規格邊界
- **必查項**:`query_material_batch(lot_id)`、`query_inspection_log(批次比對)`
- **參考案例**:INC-3012(新供應商鋼材硬度+15HRC)、INC-3028(熱處理批次差異)

## 5. 歷史案例庫(節錄)

### INC-3001 (2025-09-08, P2):T-08 球刀壽命末期造成尺寸偏移
- 症狀:某客戶醫療器材精密件,Tester 報尺寸 +0.012mm(規格 ±0.005mm),良率從 98% 掉到 72%
- 調查:`query_tool_life(T-08)` 顯示已加工 4.5 小時(預估上限 4.0 小時)、主軸負載上升 18%
- 根因:T-08 球刀後刀面磨損超出 VB 0.15mm,實際切削深度增加,造成正偏移
- 處置:立即換刀 + 過去 2 小時件全 CMM 重檢、不合規件挑出
- 預防:刀具壽命警報下修到 3.5 小時、新增主軸負載 SPC 規則(7 件均值 +10% 觸發預警)

### INC-3018 (2025-07-22, P1):夏季空調故障熱漂移
- 症狀:午後 3 台機台同時尺寸漸進性偏移,X 軸每小時漂移 +3μm
- 調查:`query_environment(today)` 廠房溫度從 23°C 升到 28°C、空調主機警報、`query_cnc_status` 顯示主軸溫度上升
- 根因:精密加工區空調冷媒外洩,廠房溫度上升 5°C,機台熱補償未涵蓋這個範圍
- 處置:立即聯絡空調廠商 + 暫停精密件加工
- 預防:增加廠房溫度 ±2°C 報警、建立熱補償重新校正 SOP

### INC-3007 (2025-11-04, P3):夾具拆洗後沒回校位
- 症狀:夾具拆洗保養後第一批件全部 X 軸 -0.08mm 偏移,後續批件正常
- 調查:`query_fixture_log(F-02)` 顯示 8:00 拆洗、8:30 上機,沒有校位紀錄;`query_recent_changes` 確認沒有 NC 改動
- 根因:技師拆洗後重新安裝沒做標準件試切校正,定位銷與機台座標偏差
- 處置:全部召回重工
- 預防:夾具拆裝 SOP 增加「上機後必做標準件試切」步驟並記錄

### INC-3025 (2026-01-15, P2):夜班改 offset 沒登記
- 症狀:早班接班發現整批件 Z 軸 -0.015mm 偏移、表面粗糙度劣化
- 調查:`query_recent_changes(M-07, last_24h)` 顯示昨晚 23:42 有 Z 偏移異動,操作者代碼 N03;早班操作者不知情
- 根因:夜班發現尺寸超差但沒走標準調機流程,直接改 Z 偏移後沒登記沒交班
- 處置:整批召回,Z 偏移恢復原值
- 預防:鎖定夜班 offset 修改權限,要主管授權;交班表強制登記異常處置

### INC-3012 (2025-12-19, P2):新供應商鋼材硬度差異
- 症狀:新批次工件加工後 Ra 值從 0.8 變 1.6,切削音變高,切削力增加
- 調查:`query_material_batch(LOT-NEW)` 顯示新供應商、`query_inspection_log` 比對發現硬度差異 +15HRC
- 根因:新供應商提供的鋼材硬度高於規格,造成切削參數不適用
- 處置:暫停該批生產 + 通知採購 + 重新切削參數測試
- 預防:新供應商首批必試切 + 硬度量測;材料進料增加硬度抽檢

## 6. SLA 與通報規範

| 事故等級 | 首次回應 | 結案目標 | 通報對象 |
|---|---|---|---|
| P1 全廠停線 / 客戶重大投訴 | 15 分鐘 | 4 小時 | 廠長、生管、客服 |
| P2 單機停線 / 批次 NG | 30 分鐘 | 8 小時 | 製造組長 |
| P3 趨勢警報 / 預防性 | 2 小時 | 24 小時 | 組長 + QA |

## 7. 結論範本

```
業務衝擊:
  - 受影響產品 / 客戶
  - 估計報廢數量 (PCS) + 重工工時 (人時)
  - 預估營收衝擊 (NTD) 或客戶交期影響

根因:
  - 一句話技術描述
  - 證據鏈(從刀具壽命 / 量測 / 環境 / 程式 等資料推論)

立即處置:
  - 具體動作 + 負責人 + 完成時間

預防措施:
  - SPC 規則 / 保養門檻調整
  - SOP 修訂建議
  - 知識庫補登(新增 INC-####)
  - 教育訓練(對哪一班 / 哪個工序)
```

## 8. Agent 行為守則

1. 全程繁體中文,使用組長聽得懂的話
2. 不過度堆 jargon,但保留關鍵技術詞彙(球刀、VB、SPC、Ra、G54、夾具校位、熱補償 等)
3. 呼叫工具前必須說明「為什麼要查這個」
4. 收到結果後必須說「看到什麼、推論什麼」
5. 找到強因果證據才下結論,不要憑空猜測
6. 引用歷史案例 ID,展示推理透明度
7. 結論的「業務衝擊」要量化(報廢數、工時、NTD)— 老闆看這個
